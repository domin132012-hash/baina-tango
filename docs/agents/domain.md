# Domain docs

This is a single-context repo. Project-specific docs take priority over general skill instructions.

## Read before project work

- `AGENTS.md` for repo rules and skill policy.
- `PROJECT_STATUS.md` for current progress and known risks.
- `HANDOVER.md` for project background, architecture, and recurring pitfalls.
- `AGENT_WORKLOG.md` for agent activity history.
- The active plan file for the current task, such as `TASK_PLAN.md`, `RIKA_PLAN.md`, or `SOGO_PLAN.md`.

If `CONTEXT.md` or `CONTEXT-MAP.md` is added later, read it before broad exploration. If it is absent, proceed with the project docs above.

## ADRs

Architecture decision records live under `docs/adr/`. Read ADRs that touch the area being changed before proposing architecture or workflow changes.

## Agent configuration

Agent-facing configuration for installed skills lives under `docs/agents/`.
