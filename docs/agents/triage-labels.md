# Triage labels

Use this repo's compact GitHub label vocabulary. Do not create extra canonical labels unless the user explicitly asks.

| Use | Label |
| --- | --- |
| Bug report or regression | `bug` |
| New user-facing capability | `feature` |
| Documentation-only work | `docs` |
| Internal restructuring or cleanup | `refactor` |
| High-priority work | `priority-high` |
| Blocked by missing information, external dependency, or explicit pause | `blocked` |

When a skill refers to an AFK-ready or ready-for-agent issue, treat that as a completeness requirement for the issue body. Apply the appropriate type label from this table, and only add `priority-high` or `blocked` when the work actually matches those meanings.
