## 4. 常见错误类型（Top 500 总结）

### A. shouldDisplay 过窄（领域词过度隐藏）
- 医学/法律/教育领域的常见词被错误隐藏
- 例：医療、違法、医学部 在现代日语中是高频词

### B. shouldDisplay 过宽（罕见义项误展示）
- 旧感叹词、古语、佛教专名、法律细分义项被默认展示
- 例：阿弥陀（佛教）、案件 sense 2（法律细分）

### C. 中文自然度不足
- 直译 family terms：丈夫 → 孩子他爸（家庭称呼语境）
- 直译 old terms：女佣 → 家政人员（去歧视用语）
- 方言词：老大娘 → 老奶奶
- 固定寒暄语意译不准确：お疲れ様 ≠ 晚安

### D. 固定寒暄语翻译不准确
- お邪魔します → "打扰了" 而非 "失陪了"
- お疲れ様 → "辛苦了/先走了" 而非 "晚安"
- 畏まりました → "明白了（敬语）" 而非 "遵命"
- お願いいたします → "请您/拜托您了" 而非只写"拜托"

### E. usageNote 信息不足或场景不准确
- 场景描述过于笼统（"用于告别时" → "用于工作、学校等场合告别时"）
- 缺少语气/礼貌程度说明
- 民俗/宗教说明需中性化

### F. shortGloss 选择不当
- 选择了较生硬/较旧的词汇（遵命、特产、极限）
- 应优先选择自然口语常用表达（明白了、伴手礼、勉强刚好）

## 5. Prompt 应加入的规则

基于 Top 500 经验，Prompt 应明确以下规则（部分已在现有 prompt 中）：

1. **固定寒暄语特殊处理**：お邪魔します、お疲れ様、畏まりました、お願いいたします、お大事に、お世話になる、お先に 需给准确场景说明
2. **家庭称呼不要直译 wife/husband**：丈夫/妻子在称呼义项中应写"孩子他爸/孩子他妈"或"您先生"
3. **避免旧/歧视/方言用语**：女佣→家政人员，老大娘→老奶奶
4. **shortGloss 优先现代自然中文**：伴手礼而非特产，明白了而非遵命
5. **usageNote 必须写场景、礼貌程度、是否口语/敬语/古语**
6. **不确定时不编造日语例句**
7. **shouldDisplay 不能因领域标签直接隐藏**：需判断是否是现代日常高频词

## 6. QA checker 应加入的规则

1. **shouldDisplay 领域词二次检查**：医学/法律/教育领域词需人工判断是否真应该隐藏
2. **固定寒暄语专项检查**：お邪魔、お疲れ、畏まりました 等 7-8 个高频固定表达
3. **家庭称呼专项检查**：お父さん/お母さん/御主人 sense 2
4. **旧/歧视/方言用语检查**：女佣、老大娘、遵命 等
5. **shortGloss 自然度检查**：优先现代自然中文
6. **usageNote 空值检查**：需要 usageNote 的固定表达是否有说明
7. **佛教/宗教/古语/法律/医学专业** shouldDisplay 检查

## 8. usageNote 写法经验

**好的用法**：
- 说明使用场景（进入别人家、办公室或房间时）
- 说明礼貌程度（比「お願いします」更礼貌）
- 说明场合限制（工作、学校等场合告别时）
- 说明是敬语/口语/古语/专业用语
- 不确定时用中文概括，不编造日语例句

**不好的用法**：
- 太笼统（"用于告别时"）
- 缺少场景信息
- 混入对学习者无用的细节（如喷嚏咒语的民俗细节）
- 编造不自然的搭配或例句

## 10. Top 1000 前应避免重复的坑

1. **不要重复 shouldDisplay 过窄问题**：检查医学/法律/教育/生活类基础词的 shouldDisplay
2. **不要重复固定寒暄语错误**：检查 お邪魔/お疲れ/畏まりました/お願いいたします
3. **不要重复家庭称呼直译**：检查 wife/husband 义项
4. **不要重复旧/歧视/方言用语**：女佣/老大娘/遵命 类
5. **不要跳过 usageNote**：高频固定表达必须有 usageNote
6. **不要假设所有领域词都应该隐藏**：先判断是否现代日常高频
7. **不要假设机器 triage 可以替代 human review**：mark_unresolved 必须人工决策
8. **不要急于 Top 1000 provider run**：先沉淀规则，更新 prompt 和 QA checker
