## 2026-06-14 — GitHub 作为项目事实源

### 决策

从现在开始，项目事实优先级如下：

1. GitHub `main` 分支中的项目文档与代码。
2. 最新 commit / PR / issue。
3. 本地代理的交接报告。
4. 聊天记录。

### 原因

聊天记录容易过期；本地未 push 的状态外部不可见。为了让 ChatGPT、Claude、Codex 都能稳定接手，必须把关键状态写进仓库。

### 执行要求

任何重要任务结束后，都必须更新：

- `PROJECT_STATUS.md`
- `HANDOVER.md`
- 必要时更新 `TASK_PLAN.md`
- 必要时更新 `ACCEPTANCE.md`
- 必要时更新 `DECISION_LOG.md`

## 2026-06-14 — 理科不能套用其他年份页码

### 决策

每套理科卷必须重新确认三科页码范围。

禁止：

- 直接套用 2023-1 的物理/化学/生物页码范围。
- 直接套用 2024 或其他年份的页码范围。
- 没看封面页码表就开始切图。

### 原因

不同年份的理科 PDF 分科页码不一致。2023 和 2024 的三科页码范围已经确认不同，盲套页码会导致题号错位。

## 2026-06-14 — 理科答案必须来自官方正解

### 决策

理科答案来源规则：

- 有独立 `XXXX理科答案.pdf` 的年份，使用独立正解表。
- 没有独立答案文件的年份，必须先确认试卷 PDF 最后一页存在正解表，再使用该页。

禁止：

- 用 OCR 自动猜答案。
- 用 AI 识图猜答案。
- 人工解题后直接写入答案。
- 未确认正解来源就做判分。

### 原因

理科训练器是判分产品，答案错误会直接破坏用户信任。答案必须是官方来源。

## 2026-06-14 — 生产部署必须手动确认

### 决策

不得擅自 push 到 `main` 部署业务功能。

例外：

- 仅文档类更新可以直接提交到 `main`，但仍需在报告中说明。

### 原因

仓库绑定 Cloudflare Pages，push 到 `main` 会自动部署。业务代码未验收就上生产风险很高。

## 2026-06-14 — `index.html` 修改规则

### 决策

`index.html` 只能用 Python 字节替换进行最小改动，尤其是缓存号 bump。

禁止普通 Edit 工具直接改 `index.html`。

### 原因

`index.html` 中存在弯引号等字符，普通 Edit 工具容易引入匹配或编码问题。
