## 1. 通用验收标准

每次任务结束前必须满足：

- `git status --short` 已输出。
- `git diff --stat` 或等价摘要已输出。
- `node --check assets/eju.js` 通过。
- 本地页面可打开。
- 关键路径用真实浏览器或 Playwright/Claude Preview 验收。
- 如果改了图片路径，必须确认图片 `naturalWidth > 0`，且无 404。
- 如果改了缓存相关资源，必须同步 bump：
  - `index.html` 中的 `eju.js?v=`
  - `assets/eju.js` 中读取 `eju-scanned-data.json?v=` 的缓存号
- 如果 push 到 `main`，必须说明这会触发 Cloudflare Pages 部署，并得到用户明确同意。

## 2. `index.html` 修改验收

`index.html` 风险较高。

要求：

- 禁止用普通 Edit 工具直接改。
- 只能用 Python 字节替换做最小修改。
- 替换前必须 `assert b.count(old)==1`。
- 替换后检查没有引入弯引号问题。
- 修改目标通常只允许是缓存号。

标准写法：

```python
b = open('index.html', 'rb').read()
old = b'eju.js?v=旧缓存号'
new = b'eju.js?v=新缓存号'
assert b.count(old) == 1
open('index.html', 'wb').write(b.replace(old, new))
```

## 3. 数学卷验收标准

适用于 `math1/*`、`math2/*`。

必须确认：

- 套卷入口出现在数学列表中。
- 进入试卷后底部导航隐藏。
- 返回套卷列表后底部导航恢复。
- 每一页试卷图片真实加载，无 404。
- 每一页答案框 DOM 与 proto 完全一致。
- 答案 key 使用 `源页:答案框`，例如 `4:A`、`10:IJ`。
- 输入答案后 localStorage 保存成功。
- 重新渲染后答案仍保留。
- 灰色参照框、说明文字、非作答框不得误标为答案框。

数学当前状态：数学1、数学2 12 套均已完成并上线。除非用户明确要求，不要重复开发数学铺开任务。

## 4. 理科卷验收标准

适用于 `science/*`。

每套理科卷必须确认：

### 4.1 数据来源

- 已确认源 PDF 路径。
- 已确认答案来源：
  - 有独立理科正解表的年份，用独立正解表。
  - 无独立正解表的年份，必须先确认试卷 PDF 最后一页存在正解表。
- 不允许靠 OCR、模型猜测、人工解题来写正确答案。

### 4.2 页码范围

- 必须先读该套卷的封面/页码表。
- 必须确认物理、化学、生物三科的实际页码范围。
- 不允许直接套用其他年份的页码范围。

### 4.3 图片

- 每道题使用的图片存在。
- 图片裁剪不能保留明显底部页码/footer/大片空白。
- 一页两题或多题如果影响做题体验，应拆成一题一屏。
- 参考资料页，例如常数表、周期表，应作为折叠参考资料，而不是题目页。

### 4.4 交互

- 三科切换正常。
- 每道题显示实际解答番号。
- 单选按钮数量与原卷选项数量一致。
- 採点后红绿标记正确。
- 得分统计正确。
- localStorage 保存不串题。

### 4.5 本地验收

至少验收：

- 物理第一题与最后一题。
- 化学第一题与最后一题。
- 生物第一题与最后一题。
- 任意错一题后，得分能正确减少。
- 刷新/重渲染后答案仍保留。

## 5. 部署验收标准

如果用户明确同意 push 到 `main`：

1. push 前输出最终 `git status --short`。
2. push 后确认 Cloudflare Pages 部署完成。
3. 线上验证 `index.html` 中缓存号为最新。
4. 线上验证关键 JS、JSON、PNG 均返回 200。
5. 线上打开关键用户路径，确认无旧缓存。
6. 如果 CDN 一次验证失败，需要多次 curl 或强制刷新确认，不要被边缘节点最终一致性误导。
